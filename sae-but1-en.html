SAÉ 3.01 C2 - Connect
Implementing a transmission device

Group project of 4: creation of a smart connected socket. Managing a defined set of parameters directly on the socket, on the web page and on the Android app — also coded — to connect directly to the socket. Use of Django, Arduino, Python and Broker to complete this project.
Socket features:
- ON / OFF from push button (visible on site/app)
- ON/OFF button on site/app with feedback
- Scheduled ON/OFF time slots from web/app
- Temperature display from the socket sensor (web/Android), with a user-defined threshold and warning notifications (web/Android + email sent to the user from Arduino code)

What I learned

Ce projet combinait programmation embarquée, développement web, application mobile et communication IoT dans un seul appareil — une vraie montée en complexité.
J'ai appris à connecter un Arduino à un backend Django via MQTT, et à maintenir le bouton physique, la page web et l'application Android synchronisés en temps réel.
Construire le système d'alertes de température m'a appris à penser à l'aspect utilisateur de l'IoT — pas seulement faire fonctionner des capteurs, mais rendre les données utiles.
Travailler en groupe de quatre m'a aussi poussée à organiser les tâches clairement et à communiquer régulièrement.

Django Broker Arduino Python MQTT Pushbullet MIT App Inventor

SAÉ 3.02
C1 — Administer C3 — Program
Developing communicating applications

Setting up a communication service using Python code. Coding a given number of clients and routers, with one storage server (SQL) to allow a company to communicate efficiently across a network while encrypting messages in transit. The client code allows the user to choose how many routers to traverse, encrypting a message in layers (Tor-like network). Each router then decrypts one layer, until the last router transmits the message in plain text to the recipient.

What I learned

Ce projet m'a donné une introduction pratique à la cryptographie. J'ai implémenté un système de chiffrement en couches inspiré du réseau Tor, où chaque routeur déchiffre une couche jusqu'à ce que le dernier transmette le message en clair.
Écrire la logique de chiffrement et de déchiffrement en Python m'a aidée à comprendre comment les clés sont gérées et utilisées à travers plusieurs sauts.
Construire le client, les routeurs et le serveur SQL comme composants séparés m'a aussi obligée à réfléchir attentivement à la communication entre chaque partie du système.

Python MariaDB Cryptographie
