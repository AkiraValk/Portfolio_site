Reflective analysis
Challenges faced

Before the B.U.T., I had never worked with HTML or CSS. Moving from Python (which I already knew) to web logic and the DOM structure required a lot of adjustment. Network communication concepts like sockets and application protocols were completely new to me.

How I overcame it

I practised a lot outside of class, including building this portfolio. Every project served as training: SAÉ 2.03 helped me grow with Django and databases, and SAÉ 3.02 greatly improved my understanding of network communication in Python. I always try to go beyond the minimum to solidify my understanding.

Transferable skills

Knowing how to build a complete web application — from database to front-end including deployment — is directly usable professionally for internal monitoring or administration tools. The documentation and code structuring logic I learned applies to any future IT project.

Reflective analysis
Challenges faced

Cette compétence a été la plus difficile à valider. Je n'ai pratiquement jamais fait d'électronique ni d'Arduino avant le BUT. La théorie des signaux était compliquée sans bases solides en physique, et le langage C d'Arduino m'a déstabilisée. Comprendre la pratique sans maîtriser la théorie est un cercle difficile à briser.

How I overcame it

I relied on help from classmates and friends who already knew Arduino, which helped me get past the difficult parts quickly. I then deepened my understanding through personal research. Every new Arduino project builds on the last, and I can see real progress from one SAÉ to the next.

Transferable skills

Knowing how to deploy and measure a wireless network, read signals and work with protocols like MQTT are directly useful skills in telecoms. The ability to pick up a new programming language under project pressure is valuable in any technical role.

Reflective analysis
Challenges faced

The main challenge in this skill was error handling when configuring network equipment. Every error code required a deep understanding of the protocol or system in use, which meant long searches through technical documentation. Without a structured method, debugging could quickly become exhausting.

How I overcame it

I developed the habit of always researching first before touching a configuration — understanding how it works before fixing what does not. Going forward, I want to go further by documenting my own steps throughout projects, building a reusable reference base on protocols and equipment I have already encountered.

Transferable skills

The diagnostic method I developed — understand the system before acting, then document the solution — is applicable in any administration context, whether networks, servers or information systems. It is a rigorous professional technique I will use in any role.
