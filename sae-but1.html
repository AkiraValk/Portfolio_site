Projet de fin d'année mettant en oeuvre toute les compétences acquise au fil de la première année. L'objectif, Installer un petit réseau d'entreprise avec de la téléphonie, une partie Admin, une partie Utilisateur ainsi qu'une partie web. SAÉ 3.02
C1 — Administrer C3 — Programmer
Développer des applications communicantes

Mise en place d'un service de communication avec des codes python. Coder un nombre donné de clients et de routeurs, avec 1 serveur de stockage (SQL) pour permettre à une entreprise de communiquer efficacement à travers un réseau tout en chiffrant les messages lors du passage dans celui-ci. Le code client permet à l'utilisateur de choisir le nombre de routeurs traversés, pour crypter un message en chaîne (réseau Thor). Puis, à chaque passage dans un routeur, celui-ci déchiffre une couche, jusqu'à ce que le dernier routeur transmette le message en clair au destinataire.

What I learned

This project gave me a practical introduction to cryptography. I implemented a layered encryption system inspired by the Tor network, where each router decrypts one layer until the last one sends the message in plain text.
Writing the encryption and decryption logic in Python helped me understand how keys are managed and used across multiple hops.
Building the client, routers and SQL server as separate components also pushed me to think carefully about how each part of the system communicates.

SAÉ 3.03
C1 — Administrer C2 — Connecter C4 — Sécuriser C5 — Surveiller
Concevoir un réseau multisites

Mise en place d'une architecture pour une entreprise multi-sites. Création et mise en place de l'infrastructure entière, de l'adressage IP, des protocoles ainsi que des services en fonction de l'appel d'offres de l'entreprise. Trois grandes parties figuraient sur l'architecture : la partie LAN concernant les 2 sites distants, la partie Services concernant les services demandés dans l'appel d'offres, ainsi que la partie FAI qui concerne le réseau entre les deux sites distants. Ce projet était à réaliser en groupe de 3, une personne par partie, tout en réalisant une documentation détaillée pour chaque partie permettant à l'utilisateur de comprendre ce qui a été mis en place, comment, ainsi que les tests qui prouvent le fonctionnement de la partie en question.

What I learned

This was my most complete networking project so far. I designed a full multi-site infrastructure from scratch — LAN, inter-site connectivity and services — which made me think about IP addressing, routing protocols and security all at the same time.
Working from a real call for tenders format taught me how to translate business needs into technical choices, which is a skill beyond just configuring equipment.
I also had to write detailed documentation for my section, explaining what I did, why, and showing it worked — a good exercise in clarity and accountability.

GNS3 Windows Server Virtual box VLAN / VRRP / OSPF BGP
